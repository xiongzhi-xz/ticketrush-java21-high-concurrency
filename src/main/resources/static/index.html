2. 抢票主链路

是否接单 - 剩余库存 - 库存来源 Redis Hash 虚拟线程 -
推荐演示先使用 Redis Lua。下方三个扣减方案用于对比不同技术路径：Redis Lua 和 Redis 分布式锁共享同一份 Redis 预热库存；MySQL 乐观锁读取数据库库存表，因此剩余库存可能和 Redis 路径不同。
Redis Lua：主推荐链路 库存来源：Redis Hash。预热按钮会重置这份库存，用来演示原子扣减、防超卖和幂等。
Redis 分布式锁：对比链路 库存来源：同一份 Redis Hash。它和 Redis Lua 会连续扣同一份 Redis 剩余库存。
MySQL 乐观锁：数据库对比 库存来源：MySQL 库存表。它不是 Redis 预热库存，用来观察热点行版本冲突和长尾延迟。
